# プレゼンテーションデザインガイドライン

効果的なプレゼン資料を作成するためのベストプラクティス集

---

## コンテンツ設計

### CONTENT-001: 1スライド1メッセージ

**原則**: 1枚のスライドで伝えることは1つに絞る

**理由**: 複数のメッセージは聴衆の理解を妨げる

**実践方法**:
- スライドのタイトルでメッセージを明示する
- 「このスライドで伝えたいことは何か？」を常に自問する
- 複数のポイントがある場合はスライドを分割する

**良い例**:
```
タイトル: 市場は成長している
内容: 年間成長率15%（過去5年平均）
```

**悪い例**:
```
タイトル: 市場動向と競合状況と当社の戦略
内容: （3つのトピックが混在）
```

---

### CONTENT-002: 7±2の法則

**原則**: 箇条書きは3-7個まで

**理由**: 人間の短期記憶の限界（心理学者ジョージ・ミラーの研究）

**実践方法**:
- 最重要な3-5個に絞る
- 7個を超える場合はカテゴリに分ける
- または複数のスライドに分割する

**良い例**:
```
主な課題:
• 市場シェアの低下
• 顧客満足度の低下
• 競合の台頭
```

**悪い例**:
```
課題:
• 市場シェアの低下
• 顧客満足度の低下
• 競合の台頭
• 人材不足
• 設備の老朽化
• ブランド力の低下
• コスト増加
• 技術の遅れ
• 販路の縮小
```

---

### CONTENT-003: 40文字ルール

**原則**: 1スライドの文字数は40文字以内

**理由**: 読むスライドではなく見るスライドを作る

**実践方法**:
- キーワードのみを残す
- 詳細は口頭で補足
- 文章ではなく単語で表現

**良い例**:
```
課題: 市場シェア低下
原因: 競合の価格攻勢
```
（22文字）

**悪い例**:
```
当社の市場シェアが低下している原因は、
競合他社による積極的な価格攻勢によるものです。
```
（48文字）

## レイアウト設計

### LAYOUT-001: Zの法則

**原則**: 視線は左上→右上→左下→右下の順に動く

**理由**: 自然な視線の流れに沿った配置が理解しやすい

**実践方法**:
- 最重要情報を左上に配置
- 次に重要な情報を右上に配置
- 補足情報を左下に配置
- アクションやまとめを右下に配置

**レイアウト図**:
```
[1. タイトル・主要メッセージ]  →  [2. サブ情報・データ]
         ↓                              ↓
[3. 詳細・説明]              →  [4. 結論・アクション]
```

---

### LAYOUT-002: 余白の重要性

**原則**: スライドの30-40%は余白にする

**理由**: 詰め込みすぎは読みにくく、疲れる

**実践方法**:
- 要素間のスペースを十分に取る
- マージンを広めに設定
- 「もう1つ入りそう」と思っても入れない

**チェックポイント**:
- [ ] スライドの端から1cm以上の余白がある
- [ ] 要素と要素の間に十分なスペースがある
- [ ] 視覚的に「息苦しさ」を感じない

---

### LAYOUT-003: グリッドシステム

**原則**: 要素を整列させる

**理由**: 整列は視覚的な美しさと理解しやすさを生む

**実践方法**:
- PowerPointのガイド線を使用
- 左揃え、中央揃え、右揃えを統一
- 要素の高さ・幅を揃える

**整列の種類**:
- **左揃え**: 最も一般的、読みやすい
- **中央揃え**: タイトルスライドに適している
- **右揃え**: 特殊な演出以外は避ける

---

## ビジュアル設計

### VISUAL-001: 図>表>文字

**原則**: 図で表現できるなら図を使う

**理由**: ビジュアルは理解を早める（脳は画像を6万倍速く処理する）

**実践方法**:
- プロセス → フロー図
- 比較 → 表・マトリクス
- データ → グラフ
- 関係性 → ベン図・相関図

**グラフの選び方**:
| データの種類 | 推奨グラフ |
|---|---|
| 時系列の変化 | 折れ線グラフ |
| カテゴリ間の比較 | 棒グラフ |
| 構成比 | 円グラフ |
| 相関関係 | 散布図 |
| 分布 | ヒストグラム |

---

### VISUAL-002: 色の使用は3色まで

**原則**: ベース色+メイン色+アクセント色

**理由**: 色が多すぎると焦点がぼやける

**実践方法**:
- **ベース色**: 背景・余白（白、グレー）
- **メイン色**: タイトル・重要要素（企業カラー）
- **アクセント色**: 強調したい部分のみ（赤、オレンジ）

**色の心理効果**:
| 色 | 印象 | 用途 |
|---|---|---|
| 青 | 信頼、安定 | ビジネス全般 |
| 赤 | 情熱、緊急 | 警告、強調 |
| 緑 | 成長、安心 | 環境、健康 |
| 黄 | 注意、明るい | 注意喚起 |
| グレー | 中立、洗練 | 背景、テキスト |

---

### VISUAL-003: フォントは2種類まで

**原則**: 見出し用と本文用

**理由**: 統一感が生まれる

**実践方法**:
- 見出し: ゴシック体（力強い）
- 本文: ゴシック体または明朝体
- 強調: 太字、サイズ変更（フォント変更は避ける）

**フォントサイズの目安**:
- タイトル: 32-40pt
- サブタイトル: 24-28pt
- 本文: 18-24pt
- 注釈: 14-16pt

**最小サイズ**: 14pt以上（これ以下は読みにくい）

## ストーリーテリング

### STORY-001: 問題→解決の流れ

**原則**: 課題を提示してから解決策を示す

**理由**: 聴衆の関心を引き、提案の価値を高める

**基本構成**:
```
1. 現状（As-Is）
   ↓
2. 問題点（Gap）
   ↓
3. 影響（Impact）
   ↓
4. 解決策（Solution）
   ↓
5. 効果（Benefit）
   ↓
6. 実行計画（Action）
```

**例**:
```
現状: 市場シェア15%
問題: 競合に押されている
影響: 売上減少、利益圧迫
解決策: 新製品投入
効果: シェア20%回復
実行: Q2開始、6ヶ月で展開
```

---

### STORY-002: 3の法則

**原則**: 重要ポイントは3つに絞る

**理由**: 3つは記憶しやすく、説得力がある

**実践方法**:
- 重要な3点を選び、それぞれ詳しく説明
- 「3つの理由」「3つのステップ」「3つの効果」
- 4つ以上ある場合は優先度をつけて3つに絞る

**心理学的根拠**:
- 2つ: 不十分に感じる
- 3つ: ちょうど良い、説得力がある
- 4つ以上: 覚えられない

---

### STORY-003: データで裏付ける

**原則**: 主張には必ず根拠を示す

**理由**: 信頼性と説得力が増す

**実践方法**:
- 統計データ: 「市場は年間15%成長（出典: ○○調査）」
- 事例: 「A社は同様の施策で売上20%増」
- 専門家の意見: 「業界団体によると...」

**データの見せ方**:
- 数字を大きく強調
- 比較対象を明示（前年比、業界平均）
- 出典を必ず記載

**注意点**:
- データの出典が信頼できるか確認
- 古いデータは避ける（2年以内が理想）
- 都合の良いデータだけを選ばない

## チェックリスト

プレゼン資料完成前の最終チェック:

### コンテンツ
- [ ] 各スライドのメッセージは1つに絞られているか
- [ ] 箇条書きは7個以内か
- [ ] 文字数は適切か（1スライド40文字以内が理想）
- [ ] 専門用語は聴衆に合っているか

### レイアウト
- [ ] 余白は十分にあるか（30-40%）
- [ ] 要素は整列しているか
- [ ] 視線の流れは自然か（Zの法則）

### ビジュアル
- [ ] 図表は適切に使用されているか
- [ ] 色は3色以内か
- [ ] フォントは2種類以内か
- [ ] フォントサイズは14pt以上か

### ストーリー
- [ ] 問題→解決の流れになっているか
- [ ] 重要ポイントは3つに絞られているか
- [ ] データで裏付けられているか
- [ ] 次のアクションは明確か

### 全体
- [ ] タイトルスライドに日付・発表者名があるか
- [ ] ページ番号は振られているか
- [ ] 誤字脱字はないか
- [ ] 出典は記載されているか
