# 自然言語マッピング

## 日付の解釈ルール

ユーザーが期日を自然言語で指定した場合、以下のルールで YYYY-MM-DD 形式に変換する:

| 入力例 | 変換ルール |
|---|---|
| 「今日中」 | 当日の日付。さらに `priority:high` を自動付与 |
| 「明日」「明日まで」 | 翌日の日付 |
| 「今週金曜」「金曜までに」 | 今週の金曜日 (もう過ぎている場合は来週の金曜) |
| 「来週月曜」 | 来週の月曜日 |
| 「月末まで」「今月中」 | 当月の末日 |
| 「来月頭」「来月初め」 | 翌月1日 |
| 「2/15」「2月15日」 | 当年の 2月15日 (過ぎている場合は翌年) |
| 「3日後」「3日以内」 | 当日 + 3日 |
| 「1週間後」「来週」 | 当日 + 7日 |

**重要**: 日付計算は `date` コマンドを使って正確に算出すること。

## 自然言語マッピング (総合)

| ユーザー入力例 | 実行内容 |
|---|---|
| 「〇〇をタスクに追加」「〇〇やらなきゃ」 | リッチな説明文付きで Issue 作成 → Project 追加 (Todo) |
| 「〇〇を急ぎで追加」「〇〇 至急」 | Issue 作成 + `priority:high` ラベル + Project 追加 |
| 「〇〇を金曜までに」「〇〇 期限は来週月曜」 | Issue 作成 + 期日フィールド設定 + Project 追加 |
| 「〇〇のバグ」「〇〇が壊れてる」 | Issue 作成 + `type:bug` ラベル |
| 「タスク一覧」「今あるタスクは？」 | `gh issue list` でオープン Issue 一覧表示 |
| 「ボード表示」「カンバン見せて」 | `gh project item-list` でステータス別に表示 |
| 「#N を進行中に」「#N 始める」 | Project ステータスを In Progress に変更 |
| 「#N を完了」「#N 終わった」 | Issue クローズ + ステータスを Done に変更 |
| 「#N を保留に」「#N 一旦止めて」 | ステータスを Pending に変更 |
| 「急ぎのタスクは？」「何が急ぎ？」「今やるべきことは？」 | 優先度・期日で分析した一覧を表示 |
| 「マージできるタスクは？」「重複タスクある？」 | 類似タスクを分析して候補を表示 |
| 「#X と #Y をマージして」「#X と #Y をまとめて」 | タスクを統合し、重複をクローズ |
| 「#N の期日を〇〇に」「#N の締切を変更」 | Project の期日フィールドを更新 |
| 「#N の優先度を上げて」 | ラベルを `priority:high` に変更 |
